## 0. 前提準備
- [ ] `cd docs/stepdocs && npm install`

- [ ] `Record_Steps.command` からシナリオを呼び出せるようにする（record_example.js の参照先を確認）

## 1. シナリオ設計（カテゴリ別）
- [x] 起動・初期設定  
  - Start_DesktopApp.command 起動 → 設定を開く → AI/APIキー/素材キー/YouTubeパスを入力 → 保存  
  - **シナリオ雛形追加**: `stepdocs/scenarios/01_launch_and_settings.json`（ダミーURL/値。実URLを置き換えて使用）
- [x] 基本フロー（台本→素材→動画→アップロード）  
  - ショートモード ON のサンプル。セレクタは環境に合わせて置き換える  
  - **シナリオ雛形追加**: `stepdocs/scenarios/02_basic_flow.json`
- [ ] 台本生成（ショートON/OFF 細分化）  
  - テーマ選択 → ブリーフ入力 → ショート調整切替 → セクション生成 → YAML/セクション一覧確認
- [ ] 背景素材・BGM設定  
  - 素材検索ウインドウから背景取得 → 適用  
  - BGM指定（ファイル/URL）、音量/ducking設定
- [ ] 音声生成・タイムライン更新  
  - VOICEVOX 話者選択 → 音声生成 → タイムライン更新 → サマリ確認
- [ ] 動画生成  
  - ショート動画モードの ON/OFF 切替 → 動画生成 → ログ/再生確認
- [ ] YouTube アップロード  
  - 生成動画からアップロード開始 → ログ確認（成功ケース／上限到達の警告ケース）
- [ ] 過去動画の再利用  
  - 「過去の生成動画」一覧から開く/アップロード/削除
- [ ] 定期実行タスク  
  - タスク追加（カテゴリ/ショート/追加KW/間隔/開始まで/自動アップロード）→ 保存 → 今すぐ実行 → ステータス表示
